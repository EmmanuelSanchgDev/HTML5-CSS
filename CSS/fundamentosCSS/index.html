    • hijo decendiente

    • hijo directo >

    • hermano general ~ 
          p ~ figure {
        width: 100%;
    }
    • hermano adjacente + 
          img + figcaption {
        font-weight: bold;
    }

    • selector de atributos [atributo="valor"]

      • podemos seleccionar un elemencto especificando su atributo en caso de que tengamos varios y solo necesitemos uno en especifico 
                a[href="nosotros"]{

        }
      • el * selecciona al elemento que no imporat donde pero si contiene una palabra en especifico, sera seleccionado 
                a[href*="inicio"]{

        }
      • el ^ selecciona el elemento que en su atributo especificado comience con el texto definido 
                a[href^="http"]{

        }
      • el $ selecciona el elemento que en su atributo especificado termine con el texto definido 
                a[href$=".com"]{

        }
      • ~ no se en que se diferencia 
                a[class~=""]{

        }
      • el | se aplicara estilos si la palabra que buscamos tiene segido de ella un - 
                a[class|="menu"]{

        }

  9. Especificidad

    css aplica los estilos por cascada, pero si por alguna rason existe algun estilo con un peso mayos al ultimo que se declaro este se mantendra, asi funciona la especificidad

    Nivel de peso
    selector peso
    Etiquetas(p) y PseudolEmentos(:) 0,0,0,1
    Clases(.), Atributos([href*=""]) y PseudoClasses 0,0,1,0
    Id's(#) 0,1,0,0
    estilos inline(<p style="propiedad: valor;">) 1,0,0,0

  14. Caja de Bloque VS Caja de Linea

      Caja de Linea

    • solo usan el espacio necesario, dejando que mas elementos se les coloquen junto a ellos
    • no es posible modificar su -contenido- (width & height)
    • Solo se puede modificar margin y padding de -derecha e izquierda- (letf & right)

      Caja de Bloque

    • usan todo el espacio disponible, aunque su tamaño no lo requiera o modifiquemos el ancho
    • permite modificar el Modelo de caja Completo
